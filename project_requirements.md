# FarmChainX – Project Requirements

## 1. Project Overview
FarmChainX is a blockchain-enabled supply chain transparency platform designed to improve traceability, accountability, and trust in organic agricultural produce. The system records production practices, supply chain movement, storage handling, certification validation, and consumer traceability using QR codes.

---

# 2. Functional Requirements

## 2.1 Identity & Stakeholder Management
- User Registration (Farmer, Transporter, Warehouse, Certifier, Retailer, Admin)
- Secure Login & Authentication
- Role-Based Access Control
- Profile Creation & Management
- Blockchain-linked Digital Identity

## 2.2 Production Transparency
- Crop Batch Creation
- Seed Source Logging
- Fertilizer & Input Logging
- Pest Control Method Entry
- Irrigation Logging
- Harvest Date Recording

## 2.3 Supply Chain Tracking
- Batch Transfer Logging
- Shipment Creation
- Ownership Update
- Status Tracking (In Transit, Stored, Delivered)
- Timestamp & Location Recording

## 2.4 Storage & Handling Transparency
- Storage Entry Logging
- Cold Storage Status
- Duration Tracking
- Batch Handling Updates

## 2.5 Smart Compliance & Certification
- Smart Contract Trigger on Ownership Transfer
- Batch Review by Certification Authority
- Approve / Reject Mechanism
- Immutable Certification Recording

## 2.6 Consumer Transparency & AI Intelligence
- QR Code Generation
- Product Journey Display Page
- Full Traceability View
- AI-Based Advisory for Farmers
- Risk Pattern Detection

---

# 3. Non-Functional Requirements
- Secure Authentication & Data Protection
- Scalable Backend Architecture
- Responsive Web Interface
- Blockchain Immutability
- System Reliability & Availability

# 4. Technical Requirements
- Frontend: HTML, CSS, JavaScript
- Backend: Node.js + Express
- Database: MongoDB
- Blockchain: Ethereum + Solidity Smart Contracts
- AI Layer: Python (Scikit-learn)
- QR Integration: QRCode.js
